
# OrderRoom v1.6 – חבילת תיקונים

זהו "Fix Pack" שמחזיר את ה‑UI לבסיס של 1.6 ומתקן את מה שביקשת

מה בפנים
- `src/styles/fixes.css` – תיקוני עיצוב גלובליים כולל אייקונים לבנים בניווט תחתון, ריווח נכון מעליו, ומסגרות כפתורי יום אספקה
- `src/components/BottomNav.jsx` – ניווט תחתון עם אייקונים לבנים
- `src/pages/OrderPage.jsx` – סניף ו‑יום אספקה באותה שורה, שורת "שמור הזמנה | שלח הזמנה" מסודרת ולא נחתכת
- `src/pages/SupplierEdit.jsx` – מסך הגדרות ״עריכת ספק״ רספונסיבי, ללא גלילה שנחתכת וללא חיתוך של עליון/תחתון, ניהול פריטים ללא גלילה לרוחב
- `public/index.html` – דוגמה לבדיקת רינדור בסיסית. אם אתם עובדים עם בנדלר קיים השתמשו רק בקבצים שב־src

איך מיישמים
1. החליפו את הקבצים המקבילים בתיקיית הפרויקט שלכם בגרסת 1.6
2. ודאו שה־CSS `fixes.css` נטען אחרי ה־CSS הראשי שלכם
3. הריצו `npm run build` ופרסמו ל‑GitHub Pages
4. אם יש לכם שמות נתיבים שונים, קחו את הרעיונות מהקבצים והחילו אותם באנשי הפרויקט שלכם

נקודות מפתח
- ניווט תחתון בגובה קבוע עם אייקונים לבנים, ו־padding תחתון אוטומטי לתוכן כדי שלא יכסה כפתורים
- שורת פעולות הזמנה sticky מעל הניווט
- כפתור יום אספקה עם מסגרת עקבית ופוקוס ברור
- במסך עריכת ספק: קונטיינר שממלא את גובה המסך עם גלילה פנימית בלבד ורוחב 100% לפריטי ניהול
